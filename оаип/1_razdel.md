# РАЗДЕЛ 1. ОСНОВЫ АЛГОРИТМИЗАЦИИ

## Тема 1. ОБЩИЕ СВЕДЕНИЯ ОБ АЛГОРИТМАХ

**Алгоритм** — точная последовательность действий, описывающая процесс преобразования исходных данных в результат за конечное число шагов.

**Свойства алгоритма:**
- **Детерминированность (однозначность)** — каждый шаг алгоритма должен быть точно определен; при одних и тех же исходных данных алгоритм всегда приводит к одному и тому же результату
- **Результативность** — алгоритм должен приводить к решению задачи за конечное число шагов
- **Массовость** — применимость к целому классу задач, различающихся исходными данными
- **Дискретность** — разбиение процесса на последовательность отдельных шагов
- **Понятность** — алгоритм должен содержать только те операции, которые доступны исполнителю
- **Конечность** — алгоритм должен завершиться за конечное число шагов

**Способы описания алгоритмов:**
- **Словесное описание** — представление алгоритма на естественном языке
- **Графическое представление (блок-схемы)** — представление алгоритма с помощью графических символов
- **Запись на алгоритмическом языке (псевдокод)** — запись алгоритма в виде, близком к языку программирования, но без строгой формализации
- **Программная реализация** — запись алгоритма на языке программирования

**Формы представления алгоритмов:**
- **Рекурсивная форма** — алгоритм определяется через самого себя
- **Итеративная форма** — алгоритм основан на многократном повторении определенных действий

## Тема 2. РАЗНОВИДНОСТИ СТРУКТУР АЛГОРИТМОВ

**Типы вычислительных процессов:**

1. **Линейный** — последовательное выполнение операций одна за другой, без ветвлений и повторений
   - Особенности: простота, предсказуемость, ограниченная функциональность
   - Применение: простейшие задачи с фиксированной последовательностью действий

2. **Разветвляющийся** — выбор одного из альтернативных путей выполнения алгоритма на основе проверки условия
   - **Условный оператор (IF-THEN-ELSE)** — полная и неполная формы
   - **Оператор выбора (CASE)** — множественное ветвление
   - Особенности: позволяет обрабатывать разные случаи, усложняет алгоритм

3. **Циклический** — многократное повторение определенных действий
   - Особенности: экономичность записи, возможность обработки массивов данных
   - Условия завершения: счетчик, логическое условие, наступление события

**Классификация циклов:**
- **Параметрические циклы:**
  - **С предусловием (WHILE)** — проверка условия перед выполнением тела цикла
    - Особенность: может не выполниться ни разу, если условие изначально ложно
    - Схема: WHILE (условие) DO (тело цикла)
  - **С постусловием (DO-WHILE)** — проверка условия после выполнения тела цикла
    - Особенность: выполняется хотя бы один раз
    - Схема: DO (тело цикла) WHILE (условие)
  - **Со счётчиком (FOR)** — выполнение заданного количества итераций
    - Особенность: известно точное количество повторений
    - Схема: FOR параметр := начало TO конец DO (тело цикла)

- **Итерационные циклы** — повторение до достижения заданной точности
  - Особенность: количество итераций заранее не известно
  - Применение: вычисление рядов, численные методы, поиск оптимума

**Вложенные циклы:**
- Структура с несколькими уровнями повторений
- Применение: обработка многомерных массивов, комбинаторные алгоритмы
- Особенности: более высокая вычислительная сложность

**Рекуррентные соотношения в алгоритмах:**
- Определение значения через предыдущие значения
- Примеры: числа Фибоначчи, факториал, динамическое программирование

## Тема 3. СТРУКТУРНОЕ ПРОГРАММИРОВАНИЕ

**Структурное программирование** — методология разработки программ, основанная на использовании только трех базовых управляющих конструкций: следование, ветвление, цикл.

**Основные принципы:**
- Программа строится из простых управляющих структур
- Каждая структура имеет один вход и один выход
- Использование процедур и функций для декомпозиции задач
- Ограничение на использование операторов безусловного перехода (GOTO)
- Последовательное выполнение инструкций сверху вниз

**Теорема Бома-Якопини:**
- Любой алгоритм можно реализовать с использованием только трех базовых структур:
  - Следование (последовательное выполнение)
  - Ветвление (выбор)
  - Цикл (повторение)

**Проектирование:**
- **Нисходящее (top-down)** — разбиение сложной задачи на более простые подзадачи
  - Начинается с определения общей структуры
  - Последовательная детализация алгоритма
  - Преимущества: лучшая понятность, естественная декомпозиция
  - Недостатки: возможные сложности на нижних уровнях декомпозиции

- **Восходящее (bottom-up)** — создание простых модулей с последующим их объединением
  - Начинается с разработки базовых функций
  - Объединение в более сложные структуры
  - Преимущества: переиспользование кода, тестирование модулей
  - Недостатки: возможные трудности интеграции компонентов

**Преобразование неструктурированных программ в структурированные:**
- **Дублирование кода** — повторение фрагментов для устранения сложных переходов
- **Введение булевых переменных** — использование флагов для управления ветвлением
- **Использование вспомогательных процедур** — выделение сложных участков в отдельные подпрограммы
- **Реструктуризация циклов** — преобразование циклов в стандартные формы

**Достоинства структурного программирования:**
- Повышение надежности программ
- Улучшение читаемости и сопровождаемости кода
- Упрощение отладки и тестирования
- Возможность доказательства корректности программ

## Тема 4. ГРАФИЧЕСКОЕ ПРЕДСТАВЛЕНИЕ АЛГОРИТМОВ

**Блок-схема** — графическое представление алгоритма с использованием стандартных символов.

**Стандарт ГОСТ 19.701-90:**
- Разработан на основе международного стандарта ISO 5807:1985
- Определяет условные графические обозначения в схемах алгоритмов
- Устанавливает правила выполнения схем

**Основные элементы блок-схем по ГОСТ 19.701-90:**
- **Терминатор (начало/конец)** — овал
  - Обозначает начало, конец или точку прерывания алгоритма
- **Процесс (действие)** — прямоугольник
  - Отображает действие или набор действий над данными
- **Ввод/вывод данных** — параллелограмм
  - Обозначает ввод данных или вывод результатов
- **Условие (решение)** — ромб
  - Обозначает ветвление в алгоритме на основе логического условия
- **Соединитель** — малый круг
  - Обозначает соединение прерванных линий потока в пределах одной страницы
- **Межстраничный соединитель** — шестиугольник
  - Обозначает соединение прерванных линий потока между разными страницами
- **Предопределенный процесс** — прямоугольник с двойными вертикальными линиями
  - Обозначает вызов подпрограммы или стандартной функции
- **Стрелки** — направление выполнения
  - Указывают последовательность выполнения операций

**Правила построения блок-схем:**
- Блоки соединяются линиями потока (сверху вниз, слева направо)
- Внутри блоков записываются действия или условия
- При ветвлении на линиях указываются значения условия (Да/Нет)
- Сложные алгоритмы разбиваются на страницы с использованием соединителей
- Схемы должны быть наглядными и легко читаемыми

**Другие способы графического представления алгоритмов:**
- **Диаграммы деятельности UML** — расширенное представление с дополнительными возможностями
- **Диаграммы Насси-Шнейдермана** — альтернативный способ представления структурированных алгоритмов
- **Псевдокод с отступами** — текстовое представление с использованием отступов для структурирования
